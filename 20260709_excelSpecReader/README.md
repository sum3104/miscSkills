# excel-spec-reader スキル

Excel で書かれた設計書（API仕様書・テスト仕様書・画面一覧など）を、AIエージェントが
毎回使い捨ての読解スクリプトを書くことなく、省トークンかつ正確に読むためのスキルです。
一度解読したブックの書式（ヘッダー行の位置・列の意味・各シートの役割）は
「フォーマットプロファイル」として対象プロジェクト側に保存し、次回以降のセッションで
再利用します。

## 構成

```
excel-spec-reader/
├── SKILL.md                        # スキル本体（英語）: 5ステップのワークフロー
├── scripts/
│   └── xlsx_reader.py              # xlsx/xlsm リーダー（Python標準ライブラリのみ・pip不要）
└── references/
    ├── profile-format.md           # フォーマットプロファイルのスキーマと運用ルール
    └── reading-recipes.md          # タスク別レシピ（1件検索/範囲読み/式の確認/変換）
```

## 導入方法

Agent Skills オープン標準（SKILL.md + frontmatter）に従っているため、
お使いのエージェントのスキルディレクトリに `excel-spec-reader`
フォルダごとコピーしてください。

- Claude Code: `.claude/skills/excel-spec-reader/`（プロジェクト用）
  または `~/.claude/skills/excel-spec-reader/`（ユーザー共通）
- GitHub Copilot: `.github/skills/excel-spec-reader/`
  （`.claude/skills/` に置いたものも読み取れます）
- Cursor: `.cursor/skills/excel-spec-reader/`（プロジェクト用）
  または `~/.cursor/skills/excel-spec-reader/`（ユーザー共通）

## 設計上のポイント

- **省トークンの段階読み**: xlsx を直接読まず、`xlsx_reader.py` で
  「structure（シート一覧・結合セル数・ヘッダー行推測）→ search（キーワードで
  行を特定）→ extract（シート・行・列を絞った TSV）」の順に読みます。
  出力には文字数上限（既定20,000字）があり、超過時は絞り込み方法を案内します。
- **日本の Excel 仕様書特有の課題に対応**:
  - 結合セルは左上（anchor）の値を結合範囲全体に充填して出力（`--raw` で無効化）
  - 日付シリアル値は書式を判定して ISO 形式で出力（`yyyy"年"m"月"d"日"` や
    和暦 `ge.m.d` などのカスタム書式・CJK組込み書式も検出）
  - ふりがな（`rPh`）は除去し、「注文ちゅうもん」式の重複読みを防止
  - 式セルは Excel が保存時に書き込むキャッシュ値（人間が目視している値）を
    表示。`--show-formulas` で式も併記でき、導出列の把握に使えます
- **フォーマットプロファイル**: 検証済みの書式を対象プロジェクトの
  `docs/excel-formats/<名前>.json` に保存します。複数の目的のシートが混在する
  ブック（いわゆる神Excel）でも、各シートの役割（api-spec / unit-test など）を
  記録するので、次回以降は目的のシートに直行できます。プロファイルは
  エージェントが自動生成しますが、保存前にヘッダー行と列の意味を実データと
  突合し、曖昧な列はユーザーに確認します（未検証の推測は保存しません）。
- **変換はプロファイル駆動のみ**: `convert`（JSON / Markdown 出力）は検証済み
  プロファイルがないと動きません。書式の解釈が確定する前の一括変換は
  「自信満々に間違った」構造化データを作るためです。実ファイルのヘッダーが
  プロファイルと食い違うと警告を出し、再検証を促します（ドリフト検知）。
- **成果物の言語**: SKILL.md は英語ですが、読解結果の要約や変換出力は
  設計書の言語（日本語の設計書なら日本語）で出力します。

## 制限事項

- 旧形式の `.xls` は非対応です（.xlsx への変換を依頼します）。
  パスワード保護されたブックも読めません。
- 式は再計算しません。表示するのは最後に Excel で保存された時点の
  キャッシュ値です（Excel で普通に編集・保存されたファイルなら目視と
  一致します。Excel 以外のツールが生成したファイルではキャッシュ値が
  無いことがあり、その場合は式を `{=...}` 形式でそのまま表示します）。
- xlsm はセル値の読み取りのみ対応します（マクロは無視）。
- 図形・テキストボックス・画像の中の文字は読めません（デザイン見本
  シートなど）。セルの背景色や罫線による意味付け（「グレー行は廃止済み」
  など）も検出できないため、凡例がある場合はユーザーへの確認が必要です。
